## 3. LIMITER SYSTEMS & WEAKNESSES

### 3.1 Primary Limiter: Shoulder Accumulation

- **Issue:** High volume pressing + pulling stacks fatigue quickly in shoulders
- **Impact:** Becomes the limiter in longer workouts, typically rounds 3–5
- **Affected Movements:** Thrusters, push press, HSPU, pull-ups, overhead work
- **Biometric Signals to Track:**
  - Heart rate recovery during overhead work
  - Grip strength degradation over time
  - Shoulder internal rotation ROM pre/post workout
  - Perceived pain scale at shoulder (0–10) post-session

### 3.2 Secondary Limiter: Barbell / Dumbbell Cycling Efficiency

- **Issue:** Breaks appear mid-workout; unbroken capacity degrades in later rounds
- **Impact:** Time lost on transitions and micro-rests during loaded movements
- **Opportunity:** Improve unbroken capacity at moderate loads (95–135 lb barbell / 40–50 lb DB)
- **Biometric Signals to Track:**
  - Rep speed consistency across rounds (if trackable via accelerometer)
  - Heart rate at break points vs. unbroken sets
  - Lactate proxy (HRV recovery between sets)

### 3.3 Tertiary Limiter: Pacing Discipline

- **Issue:** Occasionally skips programmed rest. Turns RPE 8 → 9 unintentionally.
- **Impact:** Premature fatigue in later rounds. Suboptimal total output.
- **Pattern:** Tends to go out too fast or skip transitions, driven by intrinsic motivation and love for movement.
- **Biometric Signals to Track:**
  - Heart rate zone distribution per round (Zone 4 vs Zone 5 time)
  - Round split times (should be even or negative split)
  - HRV post-workout as indicator of overreaching

## 9. BIOMETRIC INTEGRATION TARGETS

### 9.1 Key Biometric Data Points to Capture

For the app to generate optimal programming, the following biometric data should be fused with this profile:

| Data Point | Source | Why It Matters |
|---|---|---|
| **Resting Heart Rate (RHR)** | Wearable (daily) | Readiness indicator. Elevated RHR = reduce volume/intensity. |
| **Heart Rate Variability (HRV)** | Wearable (morning) | Recovery status. Low HRV = prioritize Zone 2 or rest. |
| **Max Heart Rate (MHR)** | Lab test or field test | Calibrate HR zones for pacing enforcement. |
| **Heart Rate Zones** | Derived from MHR | Zone 2 (60–70%), Zone 3 (70–80%), Zone 4 (80–90%), Zone 5 (90–100%). |
| **Average HR During Workout** | Wearable (per session) | Should correlate with target RPE. |
| **Peak HR During Workout** | Wearable (per session) | Flag if exceeding Zone 5 for >3 min (overpacing signal). |
| **Time in Zone 4–5** | Wearable (per session) | Should match RPE 8–9 target. Pacing data. |
| **Sleep Duration** | Wearable (nightly) | <6 hrs = auto-reduce next day intensity. |
| **Sleep Quality Score** | Wearable (nightly) | Poor sleep = shift to recovery or skill day. |
| **Body Weight** | Scale (weekly) | Track lean mass trends. |
| **Body Fat %** | Scale/DEXA (monthly) | Primary aesthetic progress metric. |
| **Lean Muscle Mass** | DEXA or smart scale (monthly) | Confirm muscle gain trajectory. |
| **Grip Strength** | Dynamometer (weekly) | Proxy for CNS readiness and shoulder fatigue. |
| **Calories Burned** | Wearable (per session) | Validate estimated kcal output. |
| **Active Calories (Daily)** | Wearable (daily) | TDEE calculation for nutrition planning. |
| **Step Count** | Wearable (daily) | Baseline activity level. Target 8,000–12,000. |

### 9.2 Biometric-Driven Auto-Adjustments

The app should implement these rules:

```
IF HRV < personal_baseline * 0.85 THEN
  → Reduce session RPE target by 1 point
  → Swap AMRAP/Chipper for EMOM or Zone 2 treadmill
  → Flag "Recovery Day Suggested"

IF RHR > personal_baseline + 5 bpm THEN
  → Reduce total volume by 20%
  → Eliminate overhead pressing movements
  → Add extra warm-up and cooldown

IF sleep_hours < 6 THEN
  → Cap session at 25 min
  → Reduce load (step down one DB pair)
  → Prioritize mobility + light monostructural

IF peak_HR_zone5_time > 180 seconds AND target_RPE <= 8.5 THEN
  → Flag "Overpacing Detected"
  → Suggest longer transitions between stations
  → Display round split comparison

IF body_fat_trending_down AND lean_mass_stable THEN
  → Maintain current programming
  → Confirm caloric deficit is appropriate

IF body_fat_stable AND lean_mass_trending_up THEN
  → Slightly increase monostructural volume
  → Maintain DB loads
  → Confirm slight caloric surplus is intentional

IF grip_strength < personal_baseline * 0.90 THEN
  → Reduce pulling volume by 25%
  → Flag "Shoulder/Grip Fatigue — Recovery Priority"
```
